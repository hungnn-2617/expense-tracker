# Implementation Plan: Expense Tracker

**Branch**: `001-expense-tracker` | **Date**: 2026-04-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-expense-tracker/spec.md`

## Summary

Build a personal expense tracking web application using Next.js 16
(App Router) with React 19 and TypeScript. The app enables users to
record income/expense transactions, organize them by categories, view
a visual dashboard with charts (pie + bar) grouped by day/week/month,
filter and search transactions, and export filtered data to CSV.
Data is stored in SQLite via Prisma ORM for type-safe, parameterized
database access.

## Technical Context

**Language/Version**: TypeScript 5 (strict mode) on Node.js 18.17+
**Primary Dependencies**: Next.js 16.2.4, React 19.2.4, Prisma,
Recharts, Zod, date-fns, Tailwind CSS 4
**Storage**: SQLite (development via Prisma) / PostgreSQL (production)
**Testing**: ESLint + TypeScript compiler; unit tests for services
**Target Platform**: Modern web browsers (desktop + mobile responsive)
**Project Type**: Web application (single-page feel with SSR)
**Performance Goals**: Dashboard loads < 2s for 1,000 transactions
(SC-002); CSV export < 3s for 1,000 transactions (SC-005)
**Constraints**: Single user, no authentication, VND currency only,
max ~5,000 transactions
**Scale/Scope**: 5 pages (Dashboard, Transactions, New Transaction,
Edit Transaction, Categories), 2 entities, ~15 components
